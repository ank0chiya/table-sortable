# アプリケーション仕様書 (ドラッグ＆ドロップ対応・ページネーション＆グルーピング付きテーブル)

## 1. 概要
本アプリケーションは、Next.js (App Router) と React をベースに構成され、Material UI (MUI) のデザインシステムを採用したフロントエンドシステムです。
テーブルUI内に各種フォームを内包し、以下の高度な機能を統合しています。
- **階層化（グルーピング）**: データを親となるグループと、それに属するアイテムの形で分離管理・表示機構。
- **ドラッグ＆ドロップ**: 制約付き（同一グループ内のみを許可する）流麗な並べ替え機能。
- **ページネーション**: 大容量データのレンダリング負荷を下げるため、標準 `Table` にページネーションを行いつつ、グループ構造が破綻しない仕組みの導入。

## 2. 技術スタック
- **フレームワーク**: Next.js v15.x / React 19 (App Router)
- **UIライブラリ**: Material UI (MUI) `@mui/material`, `@mui/icons-material`
- **ドラッグ＆ドロップ**: `@dnd-kit/core`, `@dnd-kit/sortable`, `@dnd-kit/utilities`
- **プログラミング言語**: TypeScript
- **アーキテクチャ**: ドメイン駆動設計 (DDD) の概念を取り入れたレイヤード構造

## 3. アプリケーションアーキテクチャ
保守性と拡張性を高めるため、フロントエンドのコード (`/src`) を役割ごとに層に分離しています。

### 3.1 Domain層 (`src/domain/`)
- ビジネスのルール・事象や、対象データのスキーマを定義する層です。
- **実装コンポーネント (`models/TableRow.ts`)**:
  - `TableGroupHeader`: テーブルのグルーピングを表す「ヘッダ行」用の型定義。
  - `TableItemRow`: 実際の値を格納する「アイテム行」用の型定義（`groupId`を持ちます）。
  - `TableNode`: 上記2つをまとめる `Union` 型。Presentation層の描画における「1行」の標準データです。
  - 初期化ロジック: ダミーデータをグループ化して自動生成する関数群。

### 3.2 Application層 (`src/application/`)
- ユーザー操作（UIイベント）に伴うステート管理やユースケースを満たすためのロジック層（カスタムフック）です。
- **実装コンポーネント (`hooks/useTableForm.ts`)**:
  - `rows`: 全データの配列ステート。
  - `paginatedRows`: **【スマート・ページネーション】** ページがグループの途中で分割された場合、前ページの終端で切れたグループを検知し、自動的に `(続き)` と付与したヘッダ要素を配列の先頭に差し込みます。
  - 各種ハンドラ: テキスト入力などによる状態更新処理と、特定の制約下でのドラッグ移動処理 (`arrayMove`) を担当します。

### 3.3 Presentation層 (`src/presentation/`)
- Application層から渡される「データ」と「イベントのハンドリング先関数」を受け取り、純粋にイベントリスナーと画面の描画に専念するUIコンポーネント群です。
- **実装コンポーネント**:
  - `DraggableTable.tsx`: テーブル全体の枠組みと `dnd-kit` の Context を提供します。フッタには `TablePagination` をマウントします。
  - `TableGroupHeaderRow.tsx`: ヘッダ行 (`isHeader: true`) の場合に使われ、全カラムを統合 (`colSpan`) したラベルを描画します。
  - `DraggableTableRow.tsx`: アイテム行用コンポーネント。各フォーム要素とドラッグ用ハンドルを描画します。

### 3.4 エントリーポイント (`src/app/page.tsx`)
- 上記各層をまとめ上げる（DI: 依存注入のような役割）トップレベルのページ層です。全体のMUIテーマ (`ThemeProvider`) を適用します。

## 4. 機能仕様

### 4.1 ドラッグ＆ドロップ (D&D) 制約
- テーブルの左端にあるハンドルを掴むことで、行の並べ替えが可能です。
- **同一グループ内の制約**: アイテムは「自分が属しているグループと同じ領域」でのみ並べ替えることができます。他のグループへドラッグしてドロップしようとしても、移動先の `groupId` と自身の `groupId` が合致しないため弾かれ、元の位置に戻ります。グループのヘッダ自体もドラッグから除外されています。

### 4.2 高パフォーマンスなページネーションの実現
- **DataGridの回避**: 大容量データを全てのセルに対して `renderCell` などで展開すると、Reactの仮想DOM再構築による著しいパフォーマンス低下が発生します。これを回避するため通常の `Table` を採用し、必要なデータだけを切り出してDOMに渡す方法を取っています。
- **安全な状態分離**: `paginatedRows` に切り出されているのは描画用の一時的な状態だけであり、内部の `rows` やグループへの紐づけ情報等は Application 層に一元管理されているため、裏側のデータを破壊することなくページごとの高速表示を実現しています。

### 4.3 リアルタイム・フォーム編集
- テーブル内の入力欄はリアルタイムに変更可能で、ユーザーが操作するたびにApplication層を通して内部ステート全体が上書きされます。そのため行をドラッグ移動させても入力値は確実に追従します。
